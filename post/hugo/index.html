Building a website using Hugo.

0001-01-01

⬅️ Back to Posts ..

Building a website sure is a tough thing to do unless you are using Hugo, the fastest static site generator. Using Hugo, Make websites is now easier than ever! You just have to follow a few simple steps and make your very own Website!

Create a website using Hugo!

I wanted to share this quick tutorial of installing Hugo on my Windows 10 machine to start the journey with yall :)

Download the Hugo latest version

First thing to do is go and download the latest release of Hugo from their official download area - Hugo GitHub Releases. You are looking for the Source Code Zip file format.

With that file now on your machine you need to get Hugo installed…

Install Hugo

Take the zip file that you’ve just downloaded and extract it, I always extract to C:\Hugo\bin

Environmental Variables

We now need to set an environment variable. If you hit the Windows key on your keyboard and then search for environmental variables, the first results should be how to edit your environmental variables.

Click on Environmental Variables at the bottom of the dialog box that is open. Click on Path in the top half of the next dialog box and edit that. Now enter the path you extracted Hugo to. In my case this will be C:\Hugo\bin

Test Install

Now close the administrator cmd and open a regular one by pressing Win+R and type cmd, Now, to test the install open up a PowerShell command shell or a Command Prompt and type in hugo version

You should get back information on the version that has been installed!.

Make a new site

cd my blog hugo new site myblog The above will create a new Hugo site in a folder named myblog. (You can change myblog to any name of your choice)

Add a Theme

See themes.gohugo.io for a list of themes to consider. This blog uses the amazing Ananke theme. First, download the theme from GitHub and add it to your site’s themes directory:

cd myblog
git init
git submodule add https://github.com/theNewDynamic/gohugo-theme-ananke.git themes/ananke

Note for non-git users:

If you do not have git installed, you can download the archive of the latest version of this theme from: https://github.com/theNewDynamic/gohugo-theme-ananke/archive/master.zip Extract that .zip file to get a “gohugo-theme-ananke-master” directory. Rename that directory to “ananke”, and move it into the “themes/” directory. Then, add the theme to the site configuration:

cd myblog
echo theme = "ananke" >> config.toml

Add some content

You can manually create content files (for example as content//.) and provide metadata in them, however you can use the new command to do a few things for you (like add title and date): hugo new posts/my-first-post.md

You can also open the whole folder using VS Code (Visual Studio) by typing code . on the cmd.

Now open the myblog folder and open content folder and go into posts, you will see a .md file. Open the file using any IDE or Notepad, you will see:

---
title: "My First Post"
date: 2019-03-26T08:47:11+01:00
draft: true
---

Make sure draft: false

You can type some text underneath the - - - such as:

---
title: "My Food Blog"
date: 2019-03-26T08:47:11+01:00
draft: false
---
Hello! Welcome to my Blog :)

Adding Images, Links, Lists and Text Styles:

If you are making a blog on your webpage, You might wonder how to add a couple of things to your website such as images, text styles, lists and Links. You can check out this Markdown Cheatsheet on GitHub.

Drafts do not get deployed; once you finish a post, update the header of the post to say draft: false. More info here.

If you want to add Images to your myblog folder so you can access them right away, add all your images in the /static folder, then embed any image like: <img src="/example.png alt="Example"> </img>

Starting The Hugo Server:

Now, lets start the server using a simple command

hugo server

Customizing the theme:

Your new site already looks great, but you will want to tweak it a little before you release it to the public.

Site Configuration

Open up config.toml in VS Code: Your new site already looks great, but you will want to tweak it a little before you release it to the public.

The baseUrl Doesn’t matter if you are using a local configuration file

Building Static Pages

When you are done with adding all the content and images into your website, It’s time to build static pages. Just open the cmd and type: hugo -D Output will be in ./public directory by default.
